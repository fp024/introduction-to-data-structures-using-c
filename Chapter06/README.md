### [06-4] 계산기 프로그램 구현
* 수식을 이루는 피연산자는 한자리 숫자로만 이뤄진다고 가정
  * 여러 자릿수를 하려면 다음 연산 토큰을 만날때까지 10씩 곱해줘야할 것 같은데... Java 프로그래머를 위한 알고리즘에선는 여러자릿수 고려를 하긴 했음.

* 연산 표기법
  * 중위 표기법(infix notation)
    * 5 + 2 / 7
  * 전위 표기법(prefix notation)
    * \+ 5 / 2 7
  * 후위 표기법(postfix notation)
    * 5 2 7 / +

* 다른 예 >
  * (1 + 2) * 7
    * 전위
      * \* + 1 2 7
    * 후위
      * 1 2 + 7 *


#### 중위 표기법을 후위 표기 법으로 바꾸는 방법: 소괄호를 고려하지 않음
1. 중위 표기법의 수식을 후위 표기법의 수식으로 바꿈
2. 후위 표기법으로 바뀐 수식을 계산하여 그결과를 얻음

* 문제 06-02
    * 3 + 2 * 4
        * => 피연산자 이동: 3 2 4 
        * => 3 2 4 * +
    * 2 * 4 + 3
        * => 피연산자 이동: 2 4 3
        * =>  2 4 * 3 +

    * 2 * 1 + 3 / 2
        * => 피연산자 이동: 2 1 3 2
        * => 2 1 * 3 2 / + 


#### 중위 표기법을 후위 표기 법으로 바꾸는 방법: 소괄호를 고려함

####  중위 표기법을 후위 표기법으로 바꾸는 프로그램의 구현
* 프로젝트
	* InfixToPostfixCalc 에서 InfixToPostfix.c
		* 중위 표기법을 입력받아 내부에서 후위표기법으로 변환하여 계산하는 프로그램 

#### 후위 표기법으로 표현된 수식의 계산 방법
* 문제 06-4
	* 4 2 * 8 +
		* 4 * 2 + 8	= 16
	* 1 2 3 + * 4 /
	* (2 + 3) * 1 / 4 = 5/4 = 1.25, 정수형 나눗셈 결과는 1


#### 후위 표기법으로 표현된 수식을 계산하는 프로그램의 구현
* 피연산자는 무조건 스택으로 옮김
* 연산자를 만나면 스택에서 두개의 피연산자를 꺼내서 계산을 한다.
* 계산 결과는 다시 스택에 넣는다.
* 프로젝트
	* InfixToPostfixCalc 에서 InfixCalcualtor.c, PostCalcualtor.c
